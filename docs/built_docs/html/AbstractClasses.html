Abstract Classes module

This module provides the interfaces for creating a robot, environment, and user interface for a toy robot simulator application. All methods defined within the classes in this module must be implemented by their sub-classes that inherit from these interfaces.

class AbstractClasses.AbstractRequester(simulator)

Bases: ABC

Interface for a generic requester. Used to create concrete classes that provide the implementation for user interactions with the simulator. The requester takes requests from the user and sends the request to a simulator of type AbstractWorld to interact with its robot. It provides abstract methods for receiving a request, verifying integrity of the request, and then sending the request for onwards processing by the simulator. The implementation of these methods are provided by the concrete subclass.

Parameters:

class AbstractClasses.AbstractRobot(robot_name='Robot')

Bases: ABC

Interface for a generic mobile robot. Creates an abstract robot with basic attributes to express current and planned X/Y coordinates, orientation, and a reference name. This interface defines abstract methods for driving and steering the robot. However, the implementation of these abstract methods must be provided by the concrete classes.

Parameters:

robot_name (str) – Reference name to be assigned to instance.

class AbstractClasses.AbstractWorld(robot, world_name='World')

Bases: ABC

Interface for a generic world. Used to create concrete classes to describe simulator environments. The interface provides basic attributes to describe the world’s X/Y dimensions, origin, and reference name. The world stores information about the robot to be used with its environment and keeps track of whether the robot is placed in the world. Implementations for all abstract methods must be provided by the concrete subclass.

Parameters:

  • robot (AbstractRobot) – Robot object to be used with the world.

  • world_name (str) – Reference name to be assigned to instance.
